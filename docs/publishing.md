# Chrome ウェブストア公開ガイド

tegakari を Chrome ウェブストアに公開するための手順書です。掲載文・権限の説明は
[`docs/store-listing.md`](./store-listing.md)、プライバシーポリシーは
[`docs/privacy-policy.md`](./privacy-policy.md) を参照してください。

## 0. 事前準備チェックリスト

- [ ] `pnpm build` が成功する
- [ ] `manifest.json` の `name` / `description` / `version` が正しい
- [ ] アイコン（128x128 を含む）が表示される
- [ ] プライバシーポリシーを公開URLで参照できる状態にする
- [ ] スクリーンショット（1280x800 推奨）を1枚以上用意する
- [ ] バージョン番号を決める（初回公開は `1.0.0` 推奨）

> バージョンは `package.json` の `"version"` を編集すると、ビルド時に
> `manifest.json` に反映されます。初回公開時は `0.0.1` → `1.0.0` への変更を推奨。

## 1. デベロッパーアカウントの登録（初回のみ）

1. <https://chrome.google.com/webstore/devconsole> にアクセス
2. Googleアカウントでログイン
3. **初回のみ登録料 5 USD**（一度きり）を支払う
4. デベロッパー情報（連絡先メールなど）を登録し、メール認証を済ませる

## 2. 配布用 ZIP の作成

```bash
pnpm install
pnpm package
```

`build/chrome-mv3-prod.zip` が生成されます。これがアップロードするファイルです。

> `pnpm package` は `plasmo build --zip` のエイリアスです。

## 3. 新規アイテムの作成とアップロード

1. デベロッパーダッシュボードで **「新しいアイテム」** をクリック
2. `build/chrome-mv3-prod.zip` をアップロード
3. ストア掲載情報を入力（[`store-listing.md`](./store-listing.md) からコピペ）
   - 拡張機能名 / 簡単な説明 / 詳細な説明
   - カテゴリ: **Developer Tools**
   - 言語: **日本語**
   - スクリーンショット・アイコンをアップロード
4. **プライバシー** タブ
   - 単一用途の説明を入力
   - 各権限の正当化文を入力（`host_permissions` / `scripting` などは必須）
   - データ利用の開示で「外部送信なし」を申告
   - プライバシーポリシー URL を入力
5. **販売地域 / 公開範囲** を設定（全世界 or 限定公開でテストも可）

## 4. 審査の提出

1. 入力内容を保存
2. **「審査用に送信」** をクリック
3. 審査完了まで通常 **数時間〜数日**。`<all_urls>` や スクリーンショット撮影
   （`activeTab`）を使うため、権限の正当化が不十分だと差し戻されやすい点に注意。
4. 承認されると自動またはワンクリックで公開されます。

## 5. 公開後のアップデート手順

1. `package.json` の `version` を上げる（例: `1.0.0` → `1.0.1`）
2. `pnpm package` で ZIP を再生成
3. ダッシュボードの該当アイテムで新しい ZIP をアップロード → 審査提出

## トラブルシューティング

- **差し戻し: 権限の説明不足** → `store-listing.md` の正当化文を見直す。特に
  `<all_urls>` は「ユーザーが任意のサイトで要素を選択するため」と明記する。
- **差し戻し: プライバシーポリシー未設定** → 公開URL（GitHubのファイルURLで可）を
  プライバシータブに入力する。
- **アイコンが粗い** → 512x512 の高解像度ソースから 128x128 を書き出す。
