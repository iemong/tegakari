# Chrome ウェブストア掲載情報

Chrome ウェブストアのデベロッパーダッシュボードに入力する項目をまとめたものです。
そのままコピー＆ペーストして使えます。

## 基本情報

| 項目 | 内容 |
|------|------|
| 拡張機能名 (Name) | tegakari |
| カテゴリ (Category) | Developer Tools（デベロッパー ツール） |
| 言語 (Language) | 日本語（必要に応じて English も追加） |

## 簡単な説明 / Short description（最大132文字）

> Web要素を選択して、HTML・フレームワーク・コンポーネント情報とスクリーンショットをAIエディタ用にコピーするChrome拡張。

English:

> Pick a web element and copy its HTML, framework/component info, and a screenshot for AI editors like Claude Code & Cursor.

## 詳細な説明 / Detailed description

```
tegakari は、Webページ上の要素を選択するだけで、その要素のコンテキスト情報を
AIエディタ（Claude Code, Cursor など）にそのまま渡せる形でコピーできる
Chrome拡張機能です。

「このボタンの挙動を変えたい」「このコンポーネントを修正したい」——
そんなとき、要素をクリックするだけで以下の情報をMarkdown / JSONL形式でまとめて
クリップボードにコピーします。

■ 主な機能
- Agent Skill を同梱（`gh skill` でインストール可能）。tegakariの出力をClaude CodeなどのAIエージェントに渡すだけで、対象のソースファイルを特定し修正・検証まで行わせられる（`tegakari-fix`）。URLとリポジトリ名を答えるだけでプレフィックス設定を対話生成できるスキルも同梱（`tegakari-prefix-rules`）
- ページ上の複数要素をクリックして選択・アノテーション
- \ や矢印キーで選択対象を親・子要素へ階層移動（狙い撃ちが難しい要素も選択しやすい）
- 同一オリジン iframe 内の要素も選択（オプションで有効化。GAS の Web アプリなど iframe 描画ページ向け）
- 選択要素にピンを配置し、指示テキストを入力
- クイック指示チップ（Spacing / Color / Text / Size / Align / Remove）をピンに追加し、意図をひと目で伝わる形に整理
- 「Adjust styles」パネルでスタイル変更をその場でプレビューしながら、変更前後の値を正確に記録
- ピン同士を線で結ぶ「Relation」機能で、2つの要素にまたがる指示（例:「この2つの間隔を揃えて」）を記録
- 要素周辺のスクリーンショットを自動キャプチャ
- HTML情報（タグ・属性・テキスト）を取得
- 選択要素の実効スタイル（computed style 差分）を取得し、余白・色などの修正指示に実測値で対応
- 選択要素にマッチするCSSルール（ファイル名・セレクタ・宣言）とCSS変数の実際の値も収集し、どのファイルのどこを直せばよいかをAIに伝える
- React / Vue のコンポーネント階層・Props・State を自動検出。Svelte / SvelteKit はコンポーネント階層とソース位置を検出
- コンポーネントのソース位置（file:line）を解決して出力（dev build で取得可能な場合）
- Next.js / Nuxt / SvelteKit などのメタフレームワークを検出
- ページメタデータ（Viewport, UserAgent, Language）を収集
- 出力形式を選択してコピー — JSONL / Markdown に加え、Claude Code 向け・Cursor 向け・トークン節約用のMinimalプリセット、さらに自分のワークフロー向けのカスタムテンプレートも定義可能
- 全アノテーションのスクリーンショットを1枚の画像としてまとめてコピー
- アノテーションセットを JSON ファイル（tegakari-annotations-*.json）として書き出し / 読み込み（共有・バックアップ）
- URLパターン別のプレフィックス設定（例: [repo=my-app]）
- アノテーションをURL単位で永続化（ページ再読み込み後も復元）
- ダーク / ライトテーマ対応（Options 画面は日本語 / 英語に対応）

■ こんな方におすすめ
- AIエディタにUIの修正を依頼することが多いフロントエンドエンジニア
- 「どの要素か」を言葉で説明するのが面倒な方
- コンポーネント階層やPropsをすばやく把握したい方
- 定型カテゴリのチップで手軽に指示を出したい方（非エンジニアでもOK）

■ プライバシー
収集した情報はすべてブラウザ内（ローカル）で処理され、外部サーバーには
一切送信されません。アナリティクスや広告、トラッキングも行いません。

オープンソースで開発しています。
GitHub: https://github.com/iemong/tegakari
```

## プライバシー関連（審査で必須の入力）

- **プライバシーポリシー URL**:
  `https://github.com/iemong/tegakari/blob/main/docs/privacy-policy.md`
  （GitHub Pages や独自ドメインで公開する場合はそのURLでも可）
- **単一用途の説明 (Single purpose)**:
  > Webページ上で選択した要素のコンテキスト情報（HTML・フレームワーク・コンポーネント情報・スクリーンショット）を収集し、AIエディタで利用できる形式でクリップボードにコピーする。
- **データ利用に関する開示 (Data usage)**:
  - 「ユーザーデータを収集または使用していますか？」→ 該当するデータはすべて
    ローカル処理のみ。外部送信なし。
  - 個人情報・健康情報・金融情報・認証情報などの収集: **なし**
  - データの第三者販売: **なし**
  - 承認された用途以外での利用: **なし**

### 権限の正当化文（Permission justifications）

ダッシュボードで各権限の理由を求められた場合に貼り付けてください。

| 権限 | 正当化文 |
|------|----------|
| `activeTab` | ユーザーが拡張アイコンをクリックしたアクティブなタブ上で、要素の選択とスクリーンショット撮影を行うために必要です。 |
| `host_permissions` (`<all_urls>`) | ユーザーが任意のWebサイトで要素を選択・検査できるようにするため、全URLでのコンテンツスクリプト動作が必要です。 |
| `scripting` | ページのフレームワーク／コンポーネント情報（React Fiber, Vue 内部構造）を読み取るためのスクリプトをページのメインワールドに注入する必要があります。 |
| `storage` | アノテーション履歴・テーマ・プレフィックスルールをローカルに保存するために必要です。 |
| リモートコードの使用 (Remote code) | **使用していません**（すべてのコードはパッケージに同梱）。 |

## 必要な画像アセット

ストア掲載に必要な画像（別途用意が必要）:

| アセット | サイズ | 必須 |
|----------|--------|------|
| ストアアイコン | 128x128 PNG | ✅（`assets/icon.png` から流用可。512x512推奨ソース） |
| スクリーンショット | 1280x800 または 640x400 PNG/JPEG（最低1枚、最大5枚） | ✅ |
| 小さいプロモタイル | 440x280 PNG/JPEG | 任意 |
| マーキープロモタイル | 1400x560 PNG/JPEG | 任意 |

> スクリーンショットは、実際に拡張を使って要素を選択し、Inboxやコピー結果が
> 見える画面を撮ると審査・訴求の両面で効果的です。
